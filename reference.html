Basic Coq Tactics

Tactic Arguments Effect
intros n1 ... nn Introduce variables into the context, giving them names. n1 through nn
simpl Simplifies the goal
reflexivity Solves a goal of the form x = x.
destruct x Considers all possible constructors of an inductive data type, generating subgoals that need to be solved separately.
rewrite H Where H is of type e1 = e2: Replaces e1 in the goal with e2.
<- H Replaces e2 with e1
(H n) Where H is of type forall x, e1 = e2: Substitutes n for x in the rewrite.
apply H Uses lemma or hypothesis H to solve the goal. If H has hypotheses, adds them as new goals.
PQ in HP Allows us to conclude Q from hypotheses P -> Q and P (in logic, modus ponens).
induction x Same as destruct but adds an inductive hypothesis to inductively defined cases.
x as [n1 IH1 | n2 IH2 | ... ] Names the variables and inductive hypotheses generated by induction with the given names.
clear H1 ... Hn Removes the hypotheses H1 through Hn from the context.
inversion H If hypothesis H states that e1 = e2, where e1 and e2 are expressions that start with different constructors, then inversion H completes the current subgoal. If they start with the same constructor, it generates hypotheses relating the subterms.
unfold name Replaces the function name with its underlying definition.

Note that these are largely special cases and simplifications. Click through to the Coq documentation for a description of the tactics' general behavior and underlying theory.

Basic Automation

Tactic Effect
trivial Checks if the goal is trivially true or equivalent to a hypothesis, solves if so. Otherwise, does nothing (does not fail).
auto Tries a collection of basic tactics to solve the goal. Otherwise, does nothing (does not fail).
omega Solves arithmetic equations over natural numbers. Fails if omega cannot solve.

See Adam Chlipala's Coq Tactics Quick Reference for a guide to more advanced tactics and automation.
